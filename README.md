# 🌿 現場家計簿（草刈り・軽トラ管理）

草刈り・軽トラ作業の**収支・経費レシート・現場写真**をスマホでサッと記録し、
月ごとに**元請けさんへLINEで報告**できる、自分用の家計簿アプリです。

## できること

| タブ | 内容 |
|------|------|
| 📅 カレンダー | 月表示。日ごとに「青=収入 / 赤=経費 / 📷=写真」の印。日をタップで内訳表示 |
| 💰 収支 | 月の売上・経費・差引利益、経費のカテゴリ内訳、現場別のまとめ |
| ➕ 記録 | 収入 or 経費を選んで、金額・現場名・メモ・写真（レシート/現場）を入力 |
| 🖼 写真 | 現場写真・レシートをギャラリー表示。**Before/After比較**（作業前・後の並べ表示）も |
| 📄 報告 | 月の作業実績を **①LINEテキスト ②報告書画像 ③現場写真** で共有。**請求書（PDF）** 発行も |

その他の機能:

- **よく使う作業（定型ボタン）**: 「草刈り1日 ¥30,000」などをボタン1つで入力
- **請求書PDF**: 宛名・屋号・明細・消費税を入れて印刷→PDF保存（LINE送付可）
- **年間集計（確定申告向け）**: 年の売上・経費を月別グラフで表示、明細をCSV書き出し
- 軽トラ関連の経費（燃料・整備・車検・保険・高速代など）は最初からカテゴリ登録済み

## 使い方（スマホ）

1. デプロイしたURLをスマホのブラウザで開く
2. ブラウザメニューから「ホーム画面に追加」→ アプリのように起動できます
3. 現場で ➕記録 → 月末に 📄報告 でLINE共有

## データの保存について

- 現在、データと写真は**この端末（ブラウザ）の中**に保存されます。オフラインでも動作します。
- 機種変更・故障に備え、⚙（右上）→「バックアップを保存」でファイル書き出しができます。別端末で「復元」すれば移せます。

### クラウド同期（複数端末で同じデータ）※次の拡張予定

複数端末での自動同期は、データ保存層（`lib/db.ts`）をクラウド（例: Supabase）実装に
差し替えることでオンにできます。UI・各画面のコードはそのまま使えるよう設計しています。

## 開発

```bash
npm install
npm run dev      # http://localhost:3000（開発）
npm run build    # 静的書き出し → out/ フォルダに生成
```

`out/` の中身がそのまま公開できる静的サイトです。

## Cloudflare Pages への公開手順

このアプリは完全にクライアント動作（サーバー不要）なので、Cloudflare Pages に
静的サイトとしてそのまま公開できます。

1. Cloudflare にログイン →「Workers & Pages」→「Create」→「Pages」→ Git連携
2. このリポジトリを選択
3. ビルド設定:
   - **フレームワーク プリセット**: Next.js (Static HTML Export) ／ または None
   - **ビルドコマンド**: `npm run build`
   - **出力ディレクトリ**: `out`
4. デプロイ完了後に発行される URL をスマホで開き、「ホーム画面に追加」

技術構成: Next.js 14 (App Router / static export) / TypeScript / Tailwind CSS / IndexedDB

## 今後: クラウド同期（Cloudflare D1 + R2）

複数端末で同じデータを見られるようにする同期機能は、Cloudflare の
**D1（データベース）** と **R2（写真の保管庫）** を使って追加する想定です。
保存処理は `lib/db.ts` に集約しており、ここを D1/R2 版に差し替えることで、
各画面のコードを変えずに同期対応にできます（別ステップで実装予定）。
